# Copilot Code Review Instructions for .md Files (i.e. spec.md, design docs, etc.)

# Follow these instructions when reviewing markdown files related to project specifications, design documents, and other planning artifacts. These guidelines will help ensure that feedback is actionable, focused on design quality, and aligned with project goals.

1. Send file to design-review-agent for review using the following instructions. These instructions are designed to guide the agent in providing actionable feedback on design documents, specifications, and other markdown files related to project planning and architecture.

2. Add critical feedback as comments directly in the markdown file when possible, especially for specific sections or requirements. This allows for contextual feedback that is easy for the team to understand and act upon.

3. For major and minor feedback, compile a list of findings in a separate section at the end of the document or in a linked review document. This helps keep the main document focused while still providing comprehensive feedback.

4. When providing feedback, always reference specific sections, requirements, or design elements in the document. This makes it easier for the team to understand the context of the feedback and take appropriate action.

5. Reference expertise in Azure Function design, architecture, and best practices when providing feedback via links. This ensures that the feedback is grounded in industry standards and can help improve the overall quality of the design.

6. Add recommendations, suggestions, etc. returned from the design-review-agents as fixes the user can easily apply to the document
